预编译

https://www.cnblogs.com/Createsequence/p/16963891.html

本质

本质是 将sql语句和用户的输入进行严格分离

避免用户的输入被当做sql语句的一部分执行

原理

将SQL语句的语法结构与数据参数分离，通过占位符实现参数化查询

先设定sql查询模版语句

然后发送到数据库服务器进行编译 编译完之后等待用户的输入

将用户的输入 通通作为 预先设定好的sql查询语句的参数部分

而不会带入到查询语句当中

过程中 即使用户输入包含恶意语句 如' OR '1'='1 最后也只会被当做字符串处理

关于这个当成字符串处理

传输参数时 并不是进行简单的转义 而是被转成了二进制格式发送给服务器

这样彻底断绝了注入的可能 它们在协议层面就被标识为字符串类型的二进制数据 而不是SQL代码。

java里面的jdbc版本

当然这里只说了java 和 php 上层的事 实际上更重要的是数据库底层如何处理这些预编译的语句 如何准备接受传参 如何解析 如何再去查询

这里就相当于是java php调用了数据库的接口去实现了预编译的效果 实际的处理逻辑还是在数据库底层当中

#{} 与 ${} 是 Mybatis场景下的两种占位符

给出结论

  • #{} 对应的内容会作为 SQL 参数的一部分通过 PreparedStatement.setXXX 装入请求；
  • ${} 对应的内容会直接作为 SQL 模板的一部分，而不会视为独立的请求参数；

也就是说

#{} 是预编译参数占位符，MyBatis会将其转换为JDBC的预编译语句
